왜 선택했나?
GAS (Gameplay Ability System)
스킬 추가 시 캐릭터 클래스를 직접 수정하지 않아도 되는 구조가 필요했습니다.
GameplayTag 기반 상태 제어로 코드 결합도를 낮추고,
쿨타임·비용·이펙트를 하나의 Ability 단위로 캡슐화했습니다.
왜 선택했나?
Weapon System
다양한 무기 확장과 유지보수를 고려해 무기 시스템을 모듈화된 구조로 설계했습니다.
무기 교체, 공격 방식 변경을 데이터 기반으로 처리하여 코드 수정 없이 새로운 무기를 추가할 수 있도록 하는 것을 목표로 했습니다.
왜 선택했나?
Tag 기반 상태 관리
bool 변수로 상태를 관리하면 조건 조합이 폭발적으로 증가합니다.
GameplayTag 계층 구조로 상태 추가·삭제가 명확하고, 태그 쿼리만으로 조건 분기가 가능합니다.
왜 선택했나?
Enhanced Input System
레거시 Input Binding은 컨텍스트 전환(UI, Combat, Cutscene)이 불편합니다.
InputMappingContext를 활용해 상태별 입력 레이어를 런타임에 추가·제거하는 구조로 설계했습니다.
03 — Deep Dive

핵심 구현

3가지 핵심 시스템의 구현 방식을 코드,이미지지와 함께 설명합니다.

WeaponSystem 구조

런타임에 현재 장착중인 무기와 상태, 입력에 따라 콤보 공격을 선택합니다.

  • UK_WeaponData : 캐릭터의 무기를 카테고리 별로 등록
  • UK_StatusAnimData : 무기 부착 관련 정보와 상태 기반 TMap검색을 통한 UK_AnimData를 등록
  • UK_AnimData : 캐릭터가 출력할 몽타주를 등록
  • 플레이어의 입력 & 캐릭터의 상태에 따른 애니메이션

    • 클래스 다이어그램

Ability System 설계

ASC를 PlayerState에 배치하고, 리스폰 시에도 어빌리티·어트리뷰트가 유지되도록 설계했습니다.

  • ASC는 PlayerState, Avatar는 Character로 분리
  • 캐릭터의 스텟은 어트리뷰트셋으로 관리
  • ExecutionCalculation으로 데미지 계산적용
  • GameplayTag로 Cooldown/Cost Effect 명확히 구분
// SetByCaller를 통한 값 전달
            void AUK_CharacterBase::ChangeWeaponStat(
            const FUK_WeaponItemData* WeaponStat)
            {
              SCOPE_CYCLE_COUNTER(ChangeWeaponStat);
              UAbilitySystemComponent* ASC = GetAbilitySystemComponent();
              if (IsValid(ASC) == false)
              return;
              FGameplayEffectSpecHandle SpecHandle =
              ASC->MakeOutgoingSpec(WeaponStatEffect, 1.f, ASC->MakeEffectContext());
              // 기존 무기 효과 제거
              if (WeaponEffectHandle.IsValid())
              {
                ASC->RemoveActiveGameplayEffect(WeaponEffectHandle);
                WeaponEffectHandle.Invalidate();
              }
              if (WeaponStat == nullptr)
              return;
              SpecHandle.Data->SetSetByCallerMagnitude(
              UK_GameplayTags::Data::WeaponStat::ExtraAttackPower,
              WeaponStat->ExtraAttackPower);

  WeaponEffectHandle = ASC->ApplyGameplayEffectSpecToSelf(*SpecHandle.Data.Get());
}
// SetSetByCallerMagnitude를 통한 값값 전달
  void AUK_CharacterBase::HealStamina()
  {
    SCOPE_CYCLE_COUNTER(HealStamina);
  
    UAbilitySystemComponent* ASC = GetAbilitySystemComponent();
    if (IsValid(ASC) == false)
      return;
  
    FGameplayEffectSpecHandle SpecHandle =
      ASC->MakeOutgoingSpec(HealStaminaEffect, 1.f, ASC->MakeEffectContext());
  
    const UUK_PlayerStatusAttributeSet* Attributes =
      ASC->GetSet<UUK_PlayerStatusAttributeSet>();
  
    const float HealStaminaAmount =
      Attributes->GetMaxStamina() * 0.01f;
  
    const float StaminaAmount =
      Attributes->GetCurrentStamina();
  
    if (StaminaAmount >= Attributes->GetMaxStamina())
    {
      return;
    }
  
    SpecHandle.Data->SetSetByCallerMagnitude(
      UK_GameplayTags::Data::EndBattle::HealStamina,
      HealStaminaAmount
    );
  
    HealStaminaEffectHandle =
      ASC->ApplyGameplayEffectSpecToSelf(*SpecHandle.Data.Get());
  }

문제 상황

캐릭터와 몬스터 전투시 FPS가 급락하는 현상 발생. stat unit으로 CPU 병목 확인.
분석 방법
Profiling 도구

05 — Problem Solving

문제 해결 경험

실제 겪은 버그와 분석·해결 과정입니다.

01 간헐적으로 착지 공격이 반복적으로 수행되는 문제 Performance ▾
현상
몬스터 위에서 공중공격시 착지공격이 반복해서 수행 됨
원인 분석
Lended시 착지공격을 수행 이때 몬스터의 콜리전에 튀어서 애니메이션이 종료 되기 전 다시 착지한다면 lended에 연동된 함수가 다시 수행되는 문제
해결
Lended시 최초의 착지공격만 수행될 수 있도록 Boolean으로 판단
02 간헐적으로 어빌리티의 실행이 안되는 문제 GAS ▾
현상
애니메이션을 출력하는 어빌리티가 실행중 파쿠르와 같이 다른 애니메이션이 동시에 출력된다면 어빌리티가 실행이 안됨.
원인 분석
공격 어빌리티의 EndAbility가 실행이 안되어 이로 인해 Attacking태그를 블락하고 있는 모든 어빌리티가 실행이 안된다고 추측, Showdebug.AbilitySystem를 통해 어빌리티가 존재하는지 파악
해결
Ability의 몽타주가 어떤이유에서든 끊긴다면 어빌리티가 종료되도록 블루프린트 노드를 재배치
