왜 선택했나?
GAS (Gameplay Ability System)
스킬 추가 시 캐릭터 클래스 직접 수정 없이 확장 가능한 구조가 필요했습니다. GameplayTag 기반 상태 제어로 코드 결합도를 낮추고, 쿨타임·비용·이펙트를 하나의 Ability 단위로 캡슐화했습니다.
왜 선택했나?
Motion Matching
State Machine 방식의 Blend Tree는 상태 수 증가 시 복잡도가 급격히 높아집니다. Motion Matching은 Pose DB에서 현재 상태에 맞는 포즈를 검색하므로 이동 방향·속도·피격 방향 변화에 자연스럽게 대응합니다.
왜 선택했나?
Tag 기반 상태 관리
bool 변수로 상태를 관리하면 조건 조합이 폭발적으로 증가합니다. GameplayTag 계층 구조(State.Combat.Stunned 등)로 상태 추가·삭제가 명확하고, 태그 쿼리만으로 조건 분기가 가능해집니다.
왜 선택했나?
Enhanced Input System
레거시 Input Binding 방식은 컨텍스트 전환(UI, Combat, Cutscene)이 불편합니다. Enhanced Input의 InputMappingContext를 활용해 상태별 입력 레이어를 런타임에 추가·제거하는 구조로 설계했습니다.

03 — Deep Dive

핵심 구현

3가지 핵심 시스템의 구현 방식을 코드와 함께 설명합니다.

Motion Matching 구조

Pose Database에 캐릭터의 과거 애니메이션 포즈를 등록하고, 런타임에 현재 Trajectory(이동 궤적)·Pose Cost를 계산해 가장 자연스러운 포즈를 선택합니다.

  • Pose DB: 발 위치·속도·방향 벡터를 특징값으로 저장
  • Trajectory Goal: 입력 방향과 현재 속도 기반 예측 궤적
  • Cost Function: Pose Cost + Trajectory Cost 가중 합산
  • 채널 가중치로 상황별(전투/이동) 우선도 조정
// PoseSearch 채널 가중치 설정 예시
void UMyAnimInstance::UpdateMotionMatchingWeights()
{
    bool bInCombat = HasTag(Tag_State_Combat);

    // 전투 시 상체 동작에 가중치 증가
    FPoseSearchChannelSettings Settings;
    Settings.PoseWeight    = bInCombat ? 0.4f : 0.6f;
    Settings.TrajectoryWeight = bInCombat ? 0.6f : 0.4f;

    SetMotionMatchingChannelWeights(Settings);
}
// Trajectory 목표 생성
FPoseSearchQueryTrajectorySample
UMyAnimInstance::BuildTrajectorySample(
    float DeltaTime) const
{
    FVector Velocity   = GetOwnerVelocity();
    FVector InputDir   = GetInputVector();

    // 0.5초 뒤 예측 위치
    FVector FuturePos  = Velocity * 0.5f
                       + InputDir * 200.f;

    return { FuturePos, Velocity };
}

Ability System 설계

GAS의 AbilitySystemComponent를 PlayerState에 배치하고, 리스폰 시에도 어빌리티·어트리뷰트가 유지되도록 설계했습니다.

  • ASC는 PlayerState, Avatar는 Character로 분리
  • SetByCaller로 런타임 데미지 값을 Ability에서 직접 전달
  • GameplayTag로 Cooldown/Cost Effect 명확히 구분
  • GameplayCue 분리로 클라이언트 이펙트 독립 처리
// SetByCaller를 통한 데미지 전달
void UGA_MeleeAttack::ApplyDamage(
    AActor* Target)
{
    FGameplayEffectSpecHandle SpecHandle =
        MakeOutgoingGameplayEffectSpec(
            DamageEffectClass, GetAbilityLevel());

    // 런타임에 데미지 값 주입
    SpecHandle.Data->SetSetByCallerMagnitude(
        TAG_Data_Damage,
        BaseDamage * ComboMultiplier);

    ApplyGameplayEffectSpecToTarget(
        SpecHandle,
        UAbilitySystemGlobals::
        GetAbilitySystemComponentFromActor(Target));
}
// Tag 기반 어빌리티 실행 조건
bool UGA_MeleeAttack::CanActivateAbility(
    ...) const
{
    // Stun/Dead 상태면 사용 불가
    if (HasMatchingGameplayTag(
            TAG_State_Stunned) ||
        HasMatchingGameplayTag(
            TAG_State_Dead))
    {
        return false;
    }
    return Super::CanActivateAbility(...);
}

커스텀 Movement 처리

UCharacterMovementComponent를 상속해 상태별 이동 파라미터를 동적으로 변경하는 구조로 설계했습니다.

  • Sprint / Walk / Combat Stance별 MaxWalkSpeed 분리
  • 회전 처리: UseControllerDesiredRotation + Yaw 보간
  • 회피 도중 입력 버퍼링으로 연속 동작 지원
  • Tick 주기 조절: 일정 거리 이상 캐릭터는 업데이트 빈도 감소
// 상태 기반 이동 속도 설정
void UMyCharMovement::UpdateMovementParams()
{
    const bool bSprinting =
        HasTag(TAG_State_Sprinting);
    const bool bInCombat  =
        HasTag(TAG_State_Combat);

    if (bSprinting)
        MaxWalkSpeed = SprintSpeed;     // 600
    else if (bInCombat)
        MaxWalkSpeed = CombatWalkSpeed; // 300
    else
        MaxWalkSpeed = DefaultSpeed;    // 450

    // 전투 중 회전 즉시 반응
    bOrientRotationToMovement = !bInCombat;
    bUseControllerDesiredRotation = bInCombat;
}

05 — Problem Solving

문제 해결 경험

실제 겪은 버그와 분석·해결 과정. 이게 진짜 실무 능력입니다.

01 스킬 쿨타임이 적용되지 않는 문제 GAS ▾
현상
스킬 사용 후 쿨타임 없이 즉시 재사용 가능. HUD의 쿨타임 UI는 정상 작동.
원인 분석
ASC를 Character에 배치했을 때, 리스폰 시 캐릭터가 Destroy → Spawn되면서 ASC도 함께 초기화됨. 쿨타임 GameplayEffect가 유지되지 않았던 것. AbilitySystemComponent::GetActivatableAbilities()로 런타임 상태 확인 후 원인 특정.
해결
ASC를 PlayerState로 이동. Character는 Avatar Actor만 담당하도록 분리. InitAbilityActorInfo(PlayerState, Character)로 재연결. 리스폰 후에도 ASC·어트리뷰트 유지 확인.
02 Tick 비용이 특정 조건에서 급증하는 문제 Performance ▾
현상
일반 상태에서는 정상이나, 전투 진입 시 GameThread Tick이 2배 이상 상승.
원인 분석
dumpticks 실행 결과, AttackTraceComponent가 매 Tick SphereTrace를 실행하고 있음 발견. 전투 진입 시 무조건 Trace 활성화되는 로직 오류. Ability 활성 중에만 Trace가 동작해야 함.
해결
Ability의 ActivateAbility에서 Trace 시작, EndAbility에서 Trace 중단하도록 수정. Tick 자체를 Disable하고 필요 시 수동 호출 방식으로 전환. GameThread Tick 18.4ms → 11.4ms 감소 (38% 절감).
03 PlayerStart 위치 문제 — 캐릭터가 맵 밖에 스폰 Gameplay ▾
현상
특정 맵 로드 후 캐릭터가 월드 원점(0,0,0) 또는 맵 밖에 스폰.
원인 분석
GameMode의 FindPlayerStart가 특정 조건에서 null 반환. PlayerStart 태그 필터링 로직에서 태그 미설정 시 폴백 처리 누락.
해결
ChoosePlayerStart_Implementation 오버라이드. 유효한 PlayerStart가 없을 경우 월드 Origin 대신 지정된 Safety Spawn으로 폴백. 태그 검색 실패 시 로그 출력으로 미스 조기 탐지.
